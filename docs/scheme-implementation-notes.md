## Архитектура модулей

### 1. Библиотека элементов (`scheme_svg_elements.py`)

**Назначение:** базовые SVG-фрагменты инженерных символов по ГОСТ.

**Ключевые функции:**
- `pipe_horizontal/pipe_vertical()` — трубопроводы с направлением потока (стрелка)
- `gate_valve()` — задвижка (два треугольника)
- `strainer()` — фильтр-грязевик (ромб с перекрестием)
- `check_valve()` — обратный клапан (треугольник + упор)
- `flow_meter()` — расходомер (круг с G1/G2/G3)
- `temp_sensor()` / `pressure_sensor()` — датчики температуры/давления
- `heat_calculator()` — тепловычислитель с таблицей параметров (SVG `<rect>` + `<text>`)
- `heat_exchanger()` — теплообменник (прямоугольник с диагоналями, 4 отвода)
- `pump()` — насос (круг + треугольник)
- `valve_3way()` / `valve_2way()` — трёхходовой/двухходовой клапан
- `radiator()` — радиатор отопления (прямоугольник с вертикальными линиями)

**Вспомогательные:**
- `svg_canvas()` — корневой `<svg>` с `viewBox` и `<defs>`
- `connection_line()` — ломаная линия (`<polyline>`)
- `dashed_rect()` — пунктирная рамка зон (УУТЭ, ГВС, вентиляция)
- `text_label()` — текстовая подпись

**Стиль:**
- Чёрно-белый: `stroke="#000"`, `fill="none"`
- Линии: 1.5 px
- Шрифт: Arial 11 px
- Координаты: абсолютные (x, y)

**Критическая ошибка и исправление:**  
`text_label()` изначально возвращала пустую строку из-за неправильного многострочного `return` без скобок. Исправлено обёрткой в `()`.

### 2. ГОСТ-рамка (`scheme_gost_frame.py`)

**Назначение:** обёртка SVG-схем в чертёжный формат с рамкой и штампом.

**Функции:**
- `gost_frame_a3(content_svg, stamp_data)` — A3 альбомная (1190×842 px)
- `gost_frame_a4(content_svg, stamp_data)` — A4 книжная (595×842 px)

**Поля:**
- Внешняя рамка: отступ 5 px от края
- Внутренняя рамка: 20 px слева (подшивка), 5 px остальные
- Рабочая область: `<g transform="translate(20,5)">` для вложенного контента

**Штамп (основная надпись):**
- Размер: ~524×156 px (правый нижний угол)
- Поля: `project_number`, `object_name`, `sheet_name`, `sheet_title`, `company`, `gip`, `executor`, `inspector`, `head`, `stage`, `sheet_num`, `total_sheets`, `format`
- Сетка: горизонтали/вертикали, мелкие ячейки для листов/формата

---

### 3. Рендерер схем (`scheme_svg_renderer.py`)

**Назначение:** компоновка полных схем из элементов библиотеки.

**Архитектура:**
- Диспетчер `render_scheme(scheme_type, params)` — маршрутизация по типу схемы
- 8 функций `render_scheme_01` … `render_scheme_08`
- Каждая возвращает SVG-фрагмент (без корневого `<svg>`), оборачиваемый ГОСТ-рамкой

**Параметры (`SchemeParams`):**
- Температуры/давления: `t1`, `t2`, `p1`, `p2`
- Расходы: `m1`, `m2`
- Тепловые нагрузки: `q_heat`, `q_gwp`
- Метки: `g1_label`, `g2_label`, `g3_label`, `t1_label`, `t2_label`, `p1_label`, `p2_label`

**Размер канвы:** 1150×820 px (рабочая область внутри ГОСТ-рамки A3)

## Каталог 8 типовых конфигураций

| № | Тип | Ключевые элементы | Размер SVG | Сложность |
|---|-----|-------------------|------------|-----------|
| **1** | Зависимая простая | Трубы, задвижки, фильтры, G1/G2, датчики, радиатор, УУТЭ | ~15 КБ | Эталон |
| **2** | Зависимая + ГВС | + теплообменники 1/2 ступени, насос ГВС, G3, ХВС/ГВС | ~20 КБ | Средняя |
| **3** | Зависимая + клапан | + 3-ходовой клапан, насос на перемычке | ~17 КБ | Средняя |
| **4** | Зависимая + клапан + ГВС | Комбинация схем 2 и 3 | ~22 КБ | Средняя |
| **5** | Зависимая + клапан + ГВС + вент | + параллельная ветка вентиляции (сверху) | ~25 КБ | Высокая |
| **6** | Независимая простая | Два контура (сетевой/внутренний) через теплообменник, 2-ходовой клапан, насос ВК, подпитка G3 | ~20 КБ | Средняя |
| **7** | Независимая + ГВС | Схема 6 + ГВС из сетевого контура | ~26 КБ | Высокая |
| **8** | Независимая + ГВС + вент | Схема 7 + ветка вентиляции | ~28 КБ | Очень высокая |

### Топологические особенности

**Зависимые схемы (1–5):**
- Один контур: сеть → потребитель → сеть
- Подача (верх) и обратка (низ) — горизонтальные магистрали
- Регулирование: трёхходовой клапан + насос на перемычке между подачей и обраткой
- ГВС: врезки в подачу (ступень 2) и обратку (ступень 1)
- Вентиляция: параллельная ветка (врезка от подачи сверху, возврат в обратку)

**Независимые схемы (6–8):**
- Два контура: сетевой (верх) и внутренний (низ) соединены теплообменником
- Сетевой: подача → G1 → теплообменник → G2 → обратка
- Внутренний: насос ВК → 2-ходовой клапан → теплообменник → радиатор → подпитка G3 → насос
- ГВС: врезка из сетевого контура (справа)
- Вентиляция: параллельная ветка из сетевого контура (сверху)

## Координатная сетка и размещение

### Общие принципы
- Координаты: абсолютные (x, y) в px
- Начало отсчёта: левый верхний угол рабочей области (после `translate(20,5)` от ГОСТ-рамки)
- Горизонтальные трубы: y = const
- Вертикальные трубы: x = const

### Типичные координаты (зависимые схемы)

```
y_supply = 150–180    # Подающий трубопровод
y_return = 480–650    # Обратный трубопровод
y_bypass = ~330       # Перемычка (среднее между подачей и обраткой)

x_start = 60          # Начало (вход Т1)
x_gate1 = 150         # Задвижка подачи
x_filter1 = 230       # Фильтр подачи
x_sensor1 = 320       # Датчики подачи
x_valve3 = 420        # Трёхходовой клапан (схемы 3–5)
x_g1 = 450–520        # Расходомер G1
x_rad = 620–750       # Радиатор (вертикальная ось)
x_g2 = 790–860        # Расходомер G2
x_sensor2 = 920–990   # Датчики обратки
x_filter2 = 1010–1080 # Фильтр обратки
x_gate2 = 1090        # Задвижка обратки

# Блок ГВС (справа от основной схемы)
gwp_x = 560–750
gwp_y_top = 200–250   # Ступень 2 (от подачи)
gwp_y_bot = 300–380   # Ступень 1 (от обратки)

# Блок вентиляции (сверху)
vent_x = 200
vent_y = 30–50
```

### Независимые схемы

```
# Сетевой контур
y_net_supply = 100–150
y_net_return = 300–350

# Внутренний контур
y_int_supply = 370–450
y_int_return = 520–650

# Теплообменник (общая ось)
x_hex = 580–600

# ГВС/вентиляция — аналогично зависимым
```

## Особенности реализации

### 1. Генерация на чистых строках

**Почему не svgwrite:**
- Требование ТЗ: без внешних библиотек
- Преимущества: полный контроль, читаемость, версионирование
- Используется `.format()` вместо f-строк для больших блоков

### 2. Модульная структура

Каждая схема — отдельная функция, но с общими принципами:
- Базовая топология (трубы, арматура, датчики, УУТЭ)
- Добавление блоков (ГВС, вентиляция) через врезки
- Пунктирные зоны (`dashed_rect`) для визуального группирования
- Подписи труб и контуров

### 3. Параметризация

Все подстановки через `params: SchemeParams`:
- Температуры/давления → таблица УУТЭ и подписи датчиков
- Расходы → таблица УУТЭ
- Тепловые нагрузки → таблица УУТЭ (`Qo`, `Qgvs`)
- Метки расходомеров/датчиков → подписи элементов

### 4. Тестирование

Каждая схема проверена:
- Компиляция Python
- Рендер SVG (сохранение в `/tmp/scheme_*.svg`)
- Проверка содержимого (5 ключевых меток/элементов)
- Визуальный контроль в браузере

## Известные ограничения MVP

1. **Фиксированный каталог:** 8 конфигураций, без модульной сборки (будет в этапе 4 roadmap)
2. **Формат:** только A3 альбомная (A4 — резерв в `gost_frame_a4`)
3. **Стиль:** чёрно-белый технический, без CAD-уровня детализации
4. **Расчёты:** только визуализация, без проверки параметров
5. **Превью:** inline SVG в браузере (PDF — задача 4)
