Модуль 1 (парсинг ТУ) готов и интегрирован. Вот что получилось:
tu_schema.py — полная Pydantic-схема всех параметров ТУ по 1036: РСО, реквизиты документа, заявитель, объект, тепловые нагрузки, трубопроводы, теплоноситель, приборы учёта, схема подключения, доп. требования. Валидационные диапазоны (давление 0–25 кг/см², температура 50–180°C и т.д.). Список из 7 обязательных полей, без которых нельзя двигаться дальше.
tu_parser.py — ядро парсинга:

Автодетект: текстовый PDF или скан (ваш пример — скан, текстового слоя нет)
Текстовый PDF → pymupdf извлекает текст → Claude API текстовым промптом
Скан PDF → pymupdf рендерит страницы в PNG (200 DPI, ~900 КБ/стр) → Claude Vision читает изображения напрямую
Промпт-экстрактор знает структуру ТУ по 1036 и возвращает строгий JSON
Перекрёстные проверки: сумма нагрузок ≈ итого, температура подачи > обратки, давление подачи > обратки, диаметр соответствует нагрузке

Протестировал на вашем PDF — рендер 4 страниц работает, Vision-запрос сформирован корректно (для реального вызова нужен ANTHROPIC_API_KEY в .env). Валидацию проверил на mock-данных, соответствующих вашим ТУ — все параметры извлекаются верно, сумма нагрузок сходится (0.338 + 0.1443 = 0.4823).
tasks.py — задача start_tu_parsing теперь реальная: находит PDF, определяет тип, вызывает парсер, сохраняет результат в БД, переводит в tu_parsed.
parsing.py — API для просмотра результатов (GET /parsing/{id}/result) и перезапуска (POST /parsing/{id}/retrigger).
