Пункт 3 — Email-отправка:
email_service.py — три слоя: Jinja2-рендеринг → SMTP-отправка (SSL/STARTTLS) с вложениями → запись в email_log. Четыре высокоуровневые функции: send_info_request(), send_reminder(), send_project(), send_error_notification().
param_labels.py — справочник: machine-коды (floor_plan) → человекочитаемые названия + подсказки + маппинг на PDF-образцы. Добавить новый параметр — одна строка в словаре.
5 HTML-шаблонов — base.html (каркас), info_request (запрос документов со списком + кнопка), reminder (с эскалацией текста), project_delivery (отправка проекта), error_notification.
emails.py — API для отладки: предпросмотр любого письма в браузере (/preview/info-request), ручная отправка, лог писем.
celery_app.py — Beat-расписание: send_reminders каждый день в 10:00 МСК.
tasks.py — заглушки заменены реальными вызовами email-сервиса с ретраями и логированием.
Пункт 4 — Страница загрузки для клиента:
upload.html — самостоятельная HTML-страница (vanilla JS, без фреймворков): чек-лист документов с live-обновлением статусов, drag-and-drop загрузка с прогресс-баром, выбор категории документа, кнопка «Готово» → вызов API. Адаптивная, работает на мобильных.
main.py — роут /upload/{order_id} отдаёт страницу, статика смонтирована.
